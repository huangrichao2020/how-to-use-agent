# 示例 01：源码级学习

[English](01-source-learning.md) · [简体中文](01-source-learning.zh-CN.md)

当你希望 agent 从另一个项目学习，但不希望它盲目复制时，用这个模式。

## 原始 prompt

```text
深度研究一下 open-design 的平面设计能力 和 agentic-stack 的 .agent/结构，看看对你有什么帮助
```

## 开发者意图

这句话要求的是比较式学习，不是立刻实现。

agent 应该先看懂每个项目如何工作，再把有用部分翻译成适合当前 agent 的模式。

## 可复用版本

```text
深度研究 [项目 A] 和 [项目 B]。

对每个项目：
- 读结构，不只读 README
- 找出核心能力
- 判断哪些能复用到我们的 agent
- 判断哪些不应该复制

返回一份简短适配备忘录：
- 有用模式
- 不兼容部分
- 最小安全实验
- 可能影响的文件或模块
```

## 好的输出形状

```text
模式：
  [外部项目做得好的地方]

为什么对我们重要：
  [当前 agent 的弱点]

适配方式：
  [更小的本地版本]

不要复制：
  [重依赖、错误抽象、项目特定行为]

第一步实验：
  [一个小改动或一份设计文档]
```
