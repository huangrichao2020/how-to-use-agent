# 示例 04：归档工作成果

[English](04-archive-the-work.md) · [简体中文](04-archive-the-work.zh-CN.md)

当 agent 已经落地一个大改动的第一阶段时，用这个模式。

## 原始 prompts

```text
开始第一步吧，举一反三，小心谨慎
```

```text
你上面的改动记录 记得写工作手册，跟大版本架构设计放进同一个目录
```

```text
已归档到 wiki：queries/daily-report-2026-04-30-agent-redesign-v2-phase1-2
跟大版本架构设计手册（architectural-redesign-v2-planned）同在 queries/ 目录下。
```

## 开发者意图

第一句话授权一个小的实现步骤。第二句话防止工作只停留在聊天上下文里。第三句话确认归档路径。

## 可复用版本

```text
开始第一阶段。

保持保守：
- 尽量复用已有逻辑
- 避免大范围重构
- 保持改动可回滚
- 测试窄行为

改完后，把工作手册写到架构设计旁边。
手册需要包括：
- 改了什么
- 为什么改
- 如何验证
- 哪些东西仍然暂停
- 第二阶段下一步做什么
- 给下一个 agent 的精确文件路径

最后报告归档路径。
```

## 完成测试

代码改完不等于任务完成。

真正完成需要满足：

- 第一步能工作
- 有验证证据
- 工作手册已经写好
- 工作手册和架构设计放在相邻位置
- 下一个 agent 能从归档路径继续
